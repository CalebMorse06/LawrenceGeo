# Agent context

Working notes for AI coding agents (Claude Code, Cursor, etc.) collaborating
on this codebase. This file is auto-loaded by Claude Code via `CLAUDE.md`
and serves as the "what you need to know before writing any code" briefing.

---

## What this project is

A GeoGuessr-style web game scoped to Lawrence, KS. Players see a Street View
panorama of somewhere in town and drop a pin on a Lawrence map; 5 rounds
total, distance-based scoring, optional save to a Supabase-backed
leaderboard. Built to be forked for other towns — see `docs/FORK_FOR_YOUR_TOWN.md`.

## Stack snapshot

- **Next.js 16** App Router (note: breaking changes from Next 15; read
  `node_modules/next/dist/docs/` before touching routing or fetching APIs)
- **Tailwind CSS v4** with CSS-variable-driven palette in `app/globals.css`
- **Mapbox GL JS** for the guess map
- **Google Street View JS API** for the panorama
- **Pannellum** loaded from CDN (not the npm package — types are broken)
- **Supabase** Postgres + RLS, plus a `SECURITY DEFINER` RPC function
- **Vitest** for unit tests
- **`tsx`** for running TypeScript scripts directly

## Things to know before writing code

### 1. Auto-generated files — do NOT hand-edit

| File | Generated by |
| --- | --- |
| `lib/generatedLocations.ts` | `scripts/fetch-osm-locations.ts` |
| `lib/curatedLocations.ts` | `scripts/curate-locations.ts` |
| `lib/extendedLocations.ts` | `scripts/curate-locations.ts` |
| `supabase/migrations/0005_lock_submissions.sql` | `scripts/build-lock-migration.ts` |

If a location is wrong, fix it in `lib/vibeLocations.ts` (hand-typed)
OR adjust the scorer in `scripts/curate-locations.ts` and regenerate.
Never edit the generated files directly — they'll be overwritten on the
next run.

The full pipeline:

```bash
npm run fetch-locations                          # OSM → generatedLocations.ts
npx tsx scripts/curate-locations.ts              # → curatedLocations + extendedLocations
npx tsx scripts/build-lock-migration.ts          # → migrations/0005
```

### 2. Score curve must stay in sync across three places

The scoring formula is `round(5000 * exp(-distance_m / 500))`. It lives in:

- `lib/scoring.ts` (`roundScore`, `SCORE_DECAY_METERS`)
- `supabase/migrations/0005_lock_submissions.sql` (the `submit_game()` function)
- `lib/scoring.test.ts` (asserted ranges)

If you change the decay constant in one place, change it in all three. The
build-lock-migration script reads from `lib/scoring.ts`, so regenerating 0005
keeps the SQL in sync — but you still need to update the tests by hand.

### 3. Locations have two ID systems

- **TypeScript / client** uses string slugs (`"allen-fieldhouse"`,
  `"vibe-mass-600"`) as the `GameLocation.id` field
- **Postgres** uses a UUID primary key plus a unique `slug text` column
  that matches the TypeScript id

The `submit_game()` RPC looks up locations by slug. When you add a
location, the slug must be unique across the merged vibe + curated pool.

### 4. Mapbox resize

Mapbox measures container size at construction. When the corner mini-map
expands, the canvas doesn't redraw unless we tell it to. `components/GuessMap.tsx`
has a `ResizeObserver` that calls `map.resize()` on every container size
change — don't remove it.

### 5. SSR + URL params

`app/play/page.tsx` is a `"use client"` component. It reads `?mode=daily`
from `window.location.search` inside a `useEffect`, NOT inside a `useState`
initializer — the initializer doesn't re-run on client hydration of a
statically-prerendered page, so daily mode would be silently lost.

### 6. Supabase write path

Anonymous INSERT on `public.games` is **revoked**. The only way to write
a score is via `supabase.rpc('submit_game', {...})`. The RPC validates
every round's score against the actual location coordinates and rejects
discrepancies. If save fails with a permissions error, it's because
migration 0005 didn't apply.

### 7. Env vars

Four `NEXT_PUBLIC_*` vars are required for the app to function:
- `NEXT_PUBLIC_MAPBOX_TOKEN`
- `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY`
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`

Plus `SUPABASE_SERVICE_ROLE_KEY` for local scripts only (never deployed).

**Never** commit `.env.local`. It's gitignored. `.mcp.json` is also gitignored
because it contains the Supabase project ref.

### 8. Daily challenge timezone

`lib/daily.ts:todayInLawrence()` uses `"America/Chicago"`. The Postgres
`submit_game()` function also computes "today" in America/Chicago. If you
fork for a town in a different timezone, change both.

## Common workflows

### Add a new hand-typed location

1. Edit `lib/vibeLocations.ts`, add an `sv(...)` entry with a unique
   `vibe-` prefixed id
2. Run `npx tsx scripts/build-lock-migration.ts`
3. Apply migration 0005 to Supabase (via MCP, SQL editor, or CLI)

### Adjust the location curation

1. Edit `scripts/curate-locations.ts` (tune `FAMOUS_KEYWORDS` or scoring weights)
2. Run `npx tsx scripts/curate-locations.ts`
3. Skim the "top 15 / bottom 10 / first 10 dropped" output
4. Iterate
5. When happy: `npx tsx scripts/build-lock-migration.ts` to regenerate the
   migration

### Change the scoring curve

1. Edit `SCORE_DECAY_METERS` in `lib/scoring.ts`
2. Update the assertions in `lib/scoring.test.ts`
3. Run `npx tsx scripts/build-lock-migration.ts` (it embeds the constant
   into the SQL function)
4. Apply the new 0005 migration

### Apply migrations via Supabase MCP

Project config lives in `.mcp.json` (gitignored). To register:

```bash
claude mcp add --scope project --transport http supabase \
  "https://mcp.supabase.com/mcp?project_ref=YOUR_REF"
```

Then `/mcp` in Claude Code → supabase → authenticate (in a regular terminal,
not the IDE extension).

## Style conventions

- No comments except for the non-obvious *why* — never the *what*.
- Functional components, no class components.
- Tailwind for everything. Avoid inline `style={{}}` except for
  one-off computed values.
- Server components by default; mark `"use client"` only when needed.
- Don't use `as any` — fix the types properly. Type assertions are okay
  for browser globals (Pannellum, Google Maps) where the official types
  are wrong or missing.
- Tests live next to the file they test (`scoring.test.ts` next to
  `scoring.ts`).

## Gotchas

- **Pannellum from npm doesn't work** — the package's types claim it's
  an ES module but it isn't. Use the CDN script tag (see `Pano360Pane.tsx`).
- **Google Maps `clickToGo: false`** — keep this off, or players can
  walk to a street sign by clicking in the pano.
- **Mapbox `attributionControl` accepts only `boolean`** in the current
  types, not an options object — don't try to pass `{ compact: true }`.
- **Next 16 statically-prerenders client components** — any `useState`
  initializer that reads `window` will get the SSR value, not the
  client value. Use `useEffect` for client-only reads.
- **Fraunces font** can't combine `weight: ["500", ...]` with `axes: ["opsz"]`
  — pick one (we use the variable font default with no `weight` or `axes`).
